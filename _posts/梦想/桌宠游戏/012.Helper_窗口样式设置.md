## 窗口位置设定

SetWindowPos(GameHWnd, NULL, 0, 0, 0, 0, SWP_NOMOVE | SWP_NOSIZE | SWP_NOZORDER | SWP_FRAMECHANGED | SWP_NOACTIVATE);

- SWP_NOMOVE → 不要移动窗口
- SWP_NOSIZE → 不要改变大小
- SWP_NOZORDER → 不要改变层级
- SWP_NOACTIVATE → 不要抢走焦点
- SWP_FRAMECHANGED → 【核心】告诉系统：边框样式变了！

## 立即更新窗口

InvalidateRect(GameHWnd, NULL, true);

标记整个窗口区域为 “需要重绘”

- NULL = 整个窗口
- true = 擦除背景重画

UpdateWindow(GameHWnd);

立刻、马上、现在就重绘！不等系统下次刷新，强制立刻更新画面。

# 透明

前两步类似，关键是：

## MARGINS

MARGINS margins = bEnable ? MARGINS{ -1 } : MARGINS{ 0, 0, 0, 0 };

MARGINS = 边缘扩展结构

-1 在 MARGINS 结构里代表 “把边框延伸到整个窗口，不留任何客户区”。
效果就是：整个窗口都变成了“边框”。在启用了 DWM 的 Windows 上，边框区域可以被渲染成完全透明，于是你的窗口内容就浮在了桌面上，没有任何矩形背景。

MARGINS {0,0,0,0} → “我不要延伸边框了，恢复正常窗口形态”，即关闭透明效果。

HRESULT hr = DwmExtendFrameIntoClientArea(GameHWnd, &margins);

DwmExtendFrameIntoClientArea = 把 Windows 的玻璃窗口框架扩展到整个游戏画面里

这个只是看起来是透明的，实际鼠标点击还是在窗口上！


## 为什么 UE 本身做不到，必须用 DWM？

UE 渲染的是客户区（中间画面），窗口透明是 Windows 系统层 功能，必须调用 DwmApi 才能让窗口背景变成透明，这是 Windows 唯一官方透明方法

# 穿透

前两步类似，关键是：

## 设置穿透

NewExStyle = CurrentExStyle | WS_EX_LAYERED | WS_EX_TRANSPARENT;

- WS_EX_LAYERED = 分层窗口（允许透明 / 穿透）
- WS_EX_TRANSPARENT = 鼠标穿透

只是设置很简单，关键是关闭标签的时候需要考虑 DWM 窗口透明是否开启

DWM 设置的透明 A 和 这里的透明标签 B 是两种东西，

当你用一个普通的、没有 WS_EX_LAYERED 样式的窗口，并且调用了 DwmExtendFrameIntoClientArea(-1) 时，系统是这样处理你的窗口的：

1. 你的窗口先在一个纯黑的画布上正常绘制。
2. 然后，DWM 负责合成画面。对于非分层窗口，系统知道“这个区域是扩展的玻璃区域”，应该把后面的画面通过颜色混合的方式透出来。
3. 但是，DWM 合成时，发现你的窗口不是分层窗口，因此它没有 Alpha 通道信息。系统不知道你的像素原本是透明还是半透明。
4. 于是，系统只能采用颜色混合，即拿你窗口上的颜色和后面的画面做混合。而你的窗口背景色是什么？是纯黑。所以它就拿着黑色去和壁纸做混合，最后的结果就是一片全黑。

简单说就是：
- 不是分层窗口：系统只做颜色混合，你的黑色背景会把后面画面完全盖住，因为黑色混合进任何颜色都是黑色。
- 是分层窗口：系统会读取你的 Alpha 通道，发现背景像素的 Alpha 是 0，就直接跳过不画，露出后面画面。

所以在关闭标签的时候，如果 DWM 窗口透明已经开启，就不能够关闭标签 WS_EX_LAYERED

## 详细认识 WS_EX_LAYERED

不开 WS_EX_LAYERED 时
- 窗口绘制方式：正常绘制。系统把你的窗口当作一个普通矩形，先画背景色（通常是黑色或白色），再在上面画内容。
- 与桌面的关系：窗口内容被当作完全不透明的矩形块，覆盖在桌面上。
- DWM 透明效果：即使调用了 DwmExtendFrameIntoClientArea，系统的合成器也只会用颜色混合来处理你的窗口。结果就是背景色（通常是黑色）与桌面混合，产生纯黑或灰暗的区域。
- 视觉效果：你的宠物就像被关在一个黑色矩形盒子里，周围不是桌面，而是黑乎乎的一片。

开 WS_EX_LAYERED 时
- 窗口绘制方式：变为“分层窗口”。系统会给窗口的每个像素额外的 Alpha 通道数据，来决定该像素的透明度。
- 与桌面的关系：系统合成器不再把窗口当成一个矩形块，而是逐个像素地检查其 Alpha 值。Alpha 为 0（完全透明）的像素会被完全跳过，直接露出后面的桌面。
- DWM 透明效果：此时 DwmExtendFrameIntoClientArea 才真正生效。背景区域的像素 Alpha 为 0，系统就直接不画，露出后面的壁纸或图标。
- 视觉效果：你的宠物完美融入桌面，没有边框，没有黑边，就像“长”在壁纸上一样。

不开 WS_EX_LAYERED，窗口背景是“实心”的，会和桌面混合出黑色；开了之后，背景才是真正“透明”的，能直接看到桌面。

## 设置窗口

SetWindowPos(GameHWnd, NULL, 0, 0, 0, 0, SWP_NOMOVE | SWP_NOSIZE | SWP_NOZORDER | SWP_FRAMECHANGED | SWP_NOACTIVATE);

所有其他 FLAG 都在说同一件事：“我什么都不想改，只想让你重新计算框架。”

- SWP_NOMOVE / SWP_NOSIZE：保持窗口原来位置和大小。
- SWP_NOZORDER：保持窗口原来的层级。
- SWP_NOACTIVATE：不要激活窗口，这在桌宠场景特别重要。你不想因为改个样式就把用户的输入焦点从正在工作的窗口抢走。

# 置顶

HWND HwndInsertAfter = bTopmost ? HWND_TOPMOST : HWND_NOTOPMOST;
SetWindowPos(GameHWnd, HwndInsertAfter, 0, 0, 0, 0, SWP_NOMOVE | SWP_NOSIZE | SWP_NOACTIVATE);

- HwndInsertAfter = HWND_TOPMOST 窗口进入置顶组，始终浮在所有非置顶窗口的上面。即使你点一下别的窗口，它也不会被盖住。
- HwndInsertAfter = HWND_NOTOPMOST 窗口回到普通组，遵循正常的 Z 序规则。点击别的窗口，它就会被盖住。
