# HWND

HWND 是 Handle to a Window 的缩写。它是 Windows 操作系统里一个最核心的概念。简单说：HWND 是 Windows 分配给每个窗口的唯一“身份证号”。

把 Windows 系统想象成一个巨型公寓楼：
- 窗口：就是公寓里的一个房间。按钮、输入框、游戏画面，这些都是房间。
- HWND：就是房间的门牌号。有了这个号码，你才能找到特定的房间，然后对它进行各种操作，比如敲门、往里放东西、甚至拆掉它的门。

Windows 系统内部管理着成千上万个这样的“窗口”，每个都有一个独一无二的 HWND 编号，不会重复。

你的桌宠运行起来后，也是一个窗口。Windows 系统会给它分配一个 HWND。

我们就是找到这个 HWND 通过这个 HWND 去操作窗口



# IsWindow (HWND)

IsWindow (HWND) 是 Windows 系统原生 API

作用：问系统 —— “这个句柄，现在还对应一个活着的窗口吗？”

窗口一重建，旧 HWND 直接作废！

HWND 会经常随着窗口的操作而重建，比如，窗口化、改分辨率、改边框模式、父子窗口级更改

## HWND 不会自己变成 nullptr

窗口销毁后，你的变量里还是存着旧数字，不会自动变空，只有 IsWindow () 能告诉你它死了。

## IsWindow 是系统级判断

它不看你的缓存，直接问 Windows 内核
