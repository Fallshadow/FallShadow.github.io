# 项目设置

总结：UE 桌宠游戏项目需要打开透明通道，关闭屏幕撕裂，设置 RHI 为 DX 11

## Alpha 支持

UE5.7 Project Setting - Engine - Rendering - Default Setting - AlphaOutput : True 
让虚幻引擎在最终渲染帧里，把 Alpha 通道的数据也一并输出

## 兼容 DX 11

Project Settings > Platforms > Windows > Default RHI > DirectX 11

RHI 的全称是 Render Hardware Interface（渲染硬件接口），让 C++ 里写的同一份渲染代码，能在不同平台、不同图形 API 上都能运行

如果没有 RHI，开发者就得为每个 API 写一套完全不同的渲染代码。有了 RHI，引擎上层只调用 RHI 提供的统一接口（比如 RHICreateTexture），由 RHI 模块负责把这条指令翻译成对应平台的 API 调用

DX11 驱动非常成熟，在旧显卡、虚拟机、云桌面环境里几乎不会出幺蛾子，编译速度也快

项目不需要 DX12 特性：你的项目没用到 Nanite、光线追踪这些强依赖 DX12 的功能

## 屏幕撕裂

r.D3D11.UseAllowTearing=0

"Allow Tearing" 是 Windows 10 引入的一项显示技术，它允许游戏在与显示器刷新率不同步的情况下直接呈现帧，绕过传统的垂直同步(V-Sync)限制。

开启时 (= 1)：
- 显卡渲染完一帧就可以立刻提交显示
- 降低输入延迟
- 代价是可能出现画面撕裂

关闭时 (= 0)：
- 强制使用传统的缓冲区交换模式
- 必须等待完整的刷新周期，画面更完整
- 但会增加延迟
