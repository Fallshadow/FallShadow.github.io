# 非动态多播委托

```Cpp
// 普通多播委托（非动态）
DECLARE_MULTICAST_DELEGATE_TwoParams(FMyDelegate, int32, float);

// 声明
FMyDelegate OnSomethingHappened;

// 绑定（C++ 成员函数）
OnSomethingHappened.AddUObject(this, &AMyActor::HandleSomething);

// 触发
OnSomethingHappened.Broadcast(42, 3.14f);
```
## 多播 (Multicast)

- 可以绑定多个函数
- 触发时所有绑定的函数都会执行
- 像一个"事件通知列表"

## 委托 (Delegate)

- 本质是类型安全的函数指针
- C++ 里声明，蓝图里也可以绑定和调用


# 动态多播委托

```Cpp
DECLARE_DYNAMIC_MULTICAST_DELEGATE_TwoParams
(
    FOnActorLoadStateChanged,  // 委托类型名称
    int32, ActorIndex,         // 第1个参数类型和名称
    bool, bIsLoaded            // 第2个参数类型和名称
);
```

## 动态 (Dynamic)

- 可以序列化，能存盘
- 蓝图可以绑定它的函数/事件
- 底层通过名称查找函数，比普通委托稍慢

## UE 自带

DECLARE_DYNAMIC_MULTICAST_DELEGATE 是虚幻引擎自带的核心宏，属于它的委托（Delegates）系统

主要为了实现一个关键功能：让 C++ 定义的函数指针（委托）能在蓝图中使用和绑定，这正是 DYNAMIC 关键字的作用。你的代码里 UPROPERTY 中的 BlueprintAssignable 标签，就是靠它来实现的。
