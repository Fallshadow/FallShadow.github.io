# TSoftClassPtr

TSoftClassPtr 是 UE 的软引用智能指针，专门用来引用一个可能还没加载到内存的类。

## 本质

> TSoftClassPtr<AActor> SoftClass;

本质上是存储了一个路径，资源路径字符串

> /Game/Blueprints/BP_Enemy.BP_Enemy_C

仅此而已。它不持有实际内存，指向的资源可能在磁盘上，也可能已经加载，它不关心

## 加载

### 同步加载（会卡顿）

UClass* LoadedClass = SoftClass.LoadSynchronous();

LoadSynchronous() 会阻塞当前线程直到加载完成。简单直接，但如果在游戏主线程调用大型资产，会导致掉帧。

### 异步加载（不卡顿）

SoftClass.LoadAsync(FStreamableDelegate::CreateLambda([]()
{
    // 加载完成后的回调
    UE_LOG(LogTemp, Log, TEXT("Class loaded!"));
}));

异步版本不阻塞，加载过程中游戏继续运行，加载完成后触发回调。适合大型资产或在游戏过程中动态加载。

### 判断进度

if (SoftClass.IsValid())  // 路径有效，但可能没加载
if (SoftClass.IsPending()) // 正在异步加载中
if (SoftClass.IsNull())   // 路径为空

## 判断是否有效

if (!SoftClass) 

```Cpp

// 情况1：从未赋值（默认构造）
if (!ActorClass)  // true，为空

// 情况2：显式设为空
ActorClass = nullptr;
if (!ActorClass)  // true

// 情况3：赋了无效路径
ActorClass = TSoftClassPtr<AActor>(FSoftObjectPath(TEXT("/Game/NonExistent/Path")));
if (!ActorClass)  // true，虽然路径有值但加载会失败

// 情况4：路径存在但类不对
ActorClass = TSoftClassPtr<AActor>(FSoftObjectPath(TEXT("/Game/Blueprints/NotAnActor")));
if (!ActorClass)  // 可能 false，因为路径有效，但类型不匹配

```

# TSubclassOf

内存中的 UClass 指针，引用它的对象加载时，资源必须存在

## 配合 TSoftClassPtr

// 方式一：同步加载并转换
TSubclassOf<AActor> SubClass = Config.ActorClass.LoadSynchronous();

// 方式二：先拿到裸 UClass，再隐式转换
UClass* RawClass = Config.ActorClass.LoadSynchronous();
TSubclassOf<AActor> SubClass = RawClass;

// 方式三：用 Get() 获取（需要已加载）
TSubclassOf<AActor> SubClass = Config.ActorClass.Get();
// Get() 返回已加载的 TSubclassOf，没加载就返回空

# 区别

TSubclassOf = "这个类我已经在内存里了"

TSoftClassPtr = "我知道有个类在磁盘上的这个路径，用的时候再加载"。
