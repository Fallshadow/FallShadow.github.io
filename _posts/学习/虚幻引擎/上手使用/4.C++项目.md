# 创建工程

创建 C++ 项目而非蓝图项目，这样可以自动生成代码，省去手动配置生成项目的麻烦。

# 代码更改的调试

之后打开 C++ 项目，并且从这里运行引擎，注意使用 单纯执行而不调试 模式，这样之后更改代码，可以在 UE 中使用 Ctrl + Alt + F11 进行代码运行时编译，同样的，如果在 UE 新增代码，也只需要使用 VS 重新加载即可。

## 代码创建的最佳方案

全程使用 UE 创建代码，如果你想分类代码，在文件浏览器中新建文件夹，然后把代码新建在这个文件夹之下。如果你的 VS 没有察觉到文件变化，请关闭 UE VS， 使用 uproject 重新生成项目解决方案。如果你想移动代码文件到另外的文件夹，也一样操作，哦对了，移动之后你要注意 include 也需要更改。

# 结构体

USTRUCT(BlueprintType)

通过 UE 的反射系统，将 C++ 结构体 “暴露给蓝图”

## 初始化规则

有些时候，结构体里面的**基础类型/结构体类型成员**必须明确初始化默认值。

Unreal 的编译器有两道 “处理线”

C++ 编译阶段	编译器（Clang/MSVC）	      负责语法正确性

反射解析阶段	Unreal Header Tool (UHT)    检查蓝图/引擎反射规则

UE 分为两种大类型模式，一种是 Game 模式，一种是 引擎/插件 模式。

Game 模块 = UHT 宽松模式，Engine / Plugin 模块 = UHT 严格模式。

- 在 Game 模块中，不需要**基础类型/结构体类型成员**明确初始化默认值。引擎会自动赋予初始值
- 在 Engine / Plugin 模块，要求每个**基础类型/结构体类型成员**明确初始化默认值。否则可能会报错（比如在打包插件的时候发现没有初始化）

# 函数

## 成员函数

声明在 .h 文件中，定义在 .cpp 文件中，它们相互之间是可见的。

## 普通函数

只在 .cpp 文件中，如果成员函数要使用它们，除非它们就定义在成员函数前，否则需要提前声明。

一般这种函数都加上 static 关键字，意味着这个函数仅在当前 cpp 可见，不会被其他 cpp 调用。
