# 反射系统

普通 C++ 是静态编译型语言，编译后丢失了类型的 “元信息”（比如类名、成员名、属性类型、函数参数等）

```Cpp
class MyClass {
public:
    int Health;
    void Attack(int Damage);
};
```

编译后，CPU 只知道Health是一块内存地址，Attack是一段机器码，但程序运行时无法 “动态知道”：
- 这个类叫什么名字？
- 它有哪些成员变量 / 函数？
- Health的类型是 int 吗？Attack的参数是 int 吗？

而 UE 的核心需求（蓝图访问 C++、编辑器可视化、序列化、网络同步）都需要运行时获取类型的元信息，因此 UE 基于 C++ 扩展了一套「反射系统」，本质是：

在编译期收集 C++ 类型的元信息，运行时提供统一的接口查询 / 操作这些信息。
