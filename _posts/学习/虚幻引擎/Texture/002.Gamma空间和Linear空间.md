# Gamma 值的来源



# 起因：人眼与显示器的差异

人眼的感知：人眼对暗部变化更敏感。当亮度从 0.1 增加到 0.2，我们感觉亮度翻倍了；但物理上光强度其实只增加了 0.1。人眼是非线性的。

CRT 显示器的特性：老式 CRT 显示器的电压与亮度关系是指数关系（Gamma ≈ 2.2）。输入 0.5 的电压，实际显示的亮度只有约 0.218。这也是一种非线性。

巧合的互补：人眼的感知曲线与 CRT 的显示曲线大致相反。因此，为了在 CRT 上正确显示图像，我们预先对图像做一个反相校正（Gamma 0.45）。这个“编码-显示”的抵消过程，就是 Gamma 校正的由来。

# 颜色处理工作流

Gamma 空间和线性空间是两种不同的颜色处理工作流。

## Gamma 空间工作流

纹理（sRGB） -> 着色器计算 -> 最终像素（sRGB） -> 显示器

- 计算在 Gamma 空间中直接进行：纹理贴图（通常是 sRGB 格式）的颜色值未经转换，直接用于光照和颜色混合。
- 物理不准确：光照计算要求亮度线性叠加（如两倍亮度 = 两倍光子数）。但在 Gamma 空间下，0.5 + 0.5 并不等于物理上的 1.0，导致光照结果变暗、颜色发脏，尤其是高光区域看起来不自然。
- 优势：性能开销小（无需实时转换）；在移动端、老平台或艺术风格偏向手绘/卡通的项目中依然常用。
