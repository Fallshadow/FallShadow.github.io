# 两种格式

## ASCII 格式

ASCII 格式的 FBX 文件是一种纯文本形式的 3D 模型文件。

主要优势：
- 可读性强：可以用记事本等任何文本编辑器打开查看
- 便于编辑：可以直接修改文件内容，方便调试
- 兼容性好：在不同软件之间传输较少出现问题
- 便于版本控制：使用 Git 等工具可以清晰看到文件变化

主要缺点：
- 文件较大：同样的模型，ASCII 格式比二进制格式占用更多空间
- 加载较慢：软件需要更多时间解析文本内容
- 不适合复杂模型：对于高精度、复杂的模型效率较低。

通常在开发、调试阶段时，ASCII 格式更为实用。

## Binary（二进制）格式

二进制FBX文件是一种专为计算机优化的格式，不像 ASCII 格式那样可以用记事本打开查看。

主要优势：
- 体积更小：文件压缩率高，占用存储空间少
- 处理更快：软件加载和处理速度明显更快
- 适合大模型：能高效处理复杂、高精度的3D模型
- 内容保护：不容易被查看和修改，有助于保护创作内容

主要缺点：
- 不可直接阅读：无法用文本编辑器查看或修改内容
- 不便于版本管理：难以跟踪文件变化
- 可能有兼容问题：在不同软件间可能出现解析错误

简单来说，如果你主要关心性能和文件大小，并且不需要手动编辑文件内容，Binary是更好的选择。

## 如何选择适合的格式

实际使用中，大多数 CG 软件默认导出和读取的都是 Binary 格式的 FBX 文件。对于一般用户而言，Binary 格式通常已经满足所有工作流程的需求，它在加载速度、存储效率和大型场景处理方面表现更优。

只有在以下少数场景下，ASCII格式才更具优势：

- 你需要调试FBX文件的结构
- 正在开发自定义导入/导出工具

因此，如果你不是在从事底层开发或调试任务，选择 Binary 格式通常是更合理的选择。
