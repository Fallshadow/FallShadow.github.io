# 内置渲染管线

Unity 旧的内置渲染管线（Built-in Forward Rendering）：两个 PASS

## Base Pass（ForwardBase）

处理：
- 主方向光（Main Directional Light）
- 环境光（Ambient）
- Lightmap / SH
- 一部分逐顶点光（Vertex Lights）

## Add Pass（ForwardAdd）

- 每个额外的逐像素光源（点光、聚光等）都会多跑一次
- 是 “每灯一个 pass”（multi-pass per light）

## 问题

如果场景里有很多实时点光源，draw call 会直接爆炸，这就是它的核心问题。

- 性能不可扩展：N 个物体 × M 个灯 → N×M 次绘制
- 带宽 & CPU 压力大：draw call 非常多


# URP（Universal Render Pipeline）

Forward+（或称 Clustered Forward）

所有灯 在一个 pass 里循环计算

- 屏幕分块（tile / cluster）
- 只给当前像素分配 “相关光源列表”

# HDRP

Deferred Rendering（延迟渲染）（默认） + Clustered Lighting

几何阶段和光照阶段分离，所有光源统一在 lighting pass 里处理，非常适合大量动态光源（几十、几百个）
