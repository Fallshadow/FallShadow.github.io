- Name "Default"

    为这个渲染通道命名为 "Default"。命名可以帮助开发者识别和管理多个渲染通道。

- CGPROGRAM

    开始一个 Cg/HLSL 程序块。Cg 是一种由 NVIDIA 开发的着色器语言，Unity 支持使用 Cg 或 HLSL 编写着色器。

- #pragma vertex VSMain // 指定顶点着色器入口函数

    指定顶点着色器的入口函数为 VSMain 。顶点着色器负责处理每个顶点的数据。会被当作 Vertex Shader 的 main 来编译/链接

- #pragma pixel PSMain // 指定像素(片元)着色器入口函数

    指定片段着色器的入口函数为 PSMain 。片段着色器负责处理每个像素的数据。会被当作 Pixel(Fragment) Shader 的 main 来编译/链接

>这不是标准 HLSL/DXC 的通用写法（标准 HLSL通常在编译命令里用 -E VSMain 指入口）。这种 #pragma vertex/#pragma pixel 更常见于某些引擎/自定义 shader pipeline（比如 Unity 的 ShaderLab、或你们自己的离线编译器）用来从单文件里提取入口点。

- #pragma target 2.0

    指定着色器的目标编译级别为 2.0。这决定了着色器可以使用的功能集和硬件兼容性。较高的版本支持更多的功能，但要求更高的硬件支持。

# 各向异性

```Cpp
#if defined(ANISOTROPIC_SAMPLE)
channels = _NormalMap.SampleBias(ce_Sampler_Anisotropic, uv, _TextureSampleBias);
#else
channels = _NormalMap.SampleBias(ce_Sampler_Repeat, uv, _TextureSampleBias);
#endif
```

- ce_Sampler_Anisotropic ：各向异性过滤，斜视角纹理更清晰，成本更高
- ce_Sampler_Repeat ：地址模式 Repeat（UV 超出 0..1 会平铺），过滤方式通常是线性/三线性（看 sampler 状态具体怎么配）

# if

GPU 很多情况下是 “先算，再决定要不要用”，而不是 “先判断再算”

要主动避免重复计算，尤其是 heavy 运算（noise / texture / loop），不要依赖 if 帮你省掉它们。 early return 也不行

只有当分支条件在同一个 warp 内所有线程都一样时，分支才真正一致。

## 怎么 “保证” 分支一致？

- 用 uniform / 常量（真正只执行一条路径）（废话）
- 基于 draw call / material 参数 （ 同一个 draw call 内完全一致）
- 基于 “空间上连续的大块区域”（大概率）

## 什么是绝对避雷的？

- 基于 UV / 屏幕坐标：
  - if (uv.x > 0.5) : 一个 warp 横跨左右区域，一半 true，一半 false，条件都会变哦，直接优化成先执行再判断
- 基于 noise / texture：
  - if (noise(uv) > 0.5) ： 每个像素随机，非常差    
- 基于逐像素数据（lighting / normal 等）:
  - if (dot(N, L) > 0) ： 每个像素不同，基本发散
