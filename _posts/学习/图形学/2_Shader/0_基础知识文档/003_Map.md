# NormalMap

廉价增加表面的细节。说到增加表面的细节，最容易想到的就是增加点数、增加面数，这显然会增加细节，但是也会带来性能开销，而法线贴图，仅仅使用一张贴图采样，来伪造出细节，极大地节省了性能。

## 存储的是什么？

法线贴图存储的是 “切线空间下的扰动方向向量”，并且经过了一次数值范围压缩（编码，使用的时候需要解码）。

如果法线贴图直接存模型空间法线：贴图会是五颜六色的彩虹色。这叫做 Object Space Normal Map。

我们现在讨论的、蓝色的那种法线贴图：它存储的是 Tangent Space Normal（切线空间法线）。

使用的时候先用数值范围解码映射，在使用 TBN 转换到世界空间下得出真正的世界空间法线向量。

```HLSL
// 最终用于光照计算的 N
float3 N = normalize(mul(decodedTangentNormal, TBN));
```

所以其实解码出来的东西 就是描述了 TBN 的偏移量。

## TBN 是什么？

法线贴图定义在 切线空间。TBN 矩阵是从切线空间变换到世界空间的一个矩阵。

切线空间是由顶点法线 N 与切线 T 、副切线 B 构成的局部坐标系，目的是在一个与模型变换无关的局部空间内描述法线的变化属性，例如用在法线贴图上，因为在这个局部切线空间中，法线默认始终朝上，我们就可以统一化对法线扰动信息进行描述。若在世界空间内描述，则法线方向会受到模型面片世界空间朝向、模型旋转 & 缩放等多方面的影响，导致法线方向会变化无法在一张图里统一描述。

是一个正交矩阵

## 切线副切线和 UV 的方向关系

切线方向与 UV 的 U 轴（横向）方向一致。

副切线方向与 UV 的 V 轴（纵向）方向一致。

![alt text](Shader_003_Map_NormalMap_UVTB.png)

“ T 平行于 U，B 平行于 V ” 只是一个 初始的、局部的、默认的 几何关系。

### 变换的情况

- 施密特正交化
- 镜像 UV 与 W 分量（手性翻转）
- 顶点绘制的流程性切线（Flow Map / 各向异性）
- 法线贴图的烘焙差异（微观欺骗）

什么时候它 “不变”？  在没有任何正交化修正、没有镜像、且没有法线贴图扰动的顶点着色器输入阶段。

## 为什么法线贴图偏蓝色

基于上面对切线空间的描述可知，在没有扰动的情况下，法线方向在切线空间中都是 (0,0,1)，但考虑向量的分量有正有负，为了需要存储到图片上就需要将 -1,1 的值映射到 0,1 上，即 N * 0.5 + 0.5，那么此时默认法线值就变成了(0.5,0.5,1)，乘以255的
颜色值，就得到了蓝紫色

## 如何在 shader 中使用

N 就用 标志 NORMAL          T 就用 标志 TANGENT


```Cpp
V2FData vert(MeshData input)
{
    V2FData output;
    output.pos = UnityObjectToclipPos(input.vertex);
    output.uv = input.uv;
    output.uv2 = input.uv2;
    output.normalWs = UnityObjectToWorldNormal(input.normaloS);
    output.posWS = mul(unity_0bjectToWorld, input.vertex);
    output.pos0S =input.vertex.xyz;
    output.tangentWS = UnityObjectToWorldDir(input.tangentos);
    // 理想情况下，B 应该指向 V 轴正方向。但为了性能我们直接用叉乘构造正交基。
    // 此时我们需要知道原本的 T 和 N 构成的是左手系还是右手系。
    // 因为我们已经把 T 投影到垂直 N 的平面了，原始手性信息只能靠 tangent.w 传递。
    output.bitangentWS = cross(output,normalWS, output.tangentWS)* input.tangentOs.w;
    output.normal0S = input.normal0S;
    output.vertexColor = input.vertexColor;
    return output;
}

// 片段着色器 (Fragment Shader)
fixed4 frag (v2f i) : SV_Target 
{
    // 1. 采样并解码法线贴图（为了适配各个平台，就使用 UnpackNormal 就好）
    float3 tangentNormal = UnpackNormal(tex2D(_NormalMap, i.uv));

    // 2. 构建 TBN 矩阵 (列主序: T, B, N)
    float3x3 TBN = float3x3(i.tangentWS, i.bitangentWS, i.normalWS);

    // 3. 变换空间 (矩阵左乘向量)
    float3 worldNormal = mul(TBN, tangentNormal);

    // 4. 用 worldNormal 去算光照...
    ...
}
```
