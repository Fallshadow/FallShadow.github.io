## 是什么？

Unity Performance Reporting，即 Unity 性能报告。

这是 unity 中国团队提供的一套性能优化解决方案，它是一系列性能分析工具的集合，并将这些工具收集的数据进行可视化。

## 优点

与 unity 的内置工具和第三方性能分析工具相比
- 它不受收集数据量的限制
- 不用集成SDK，使用非侵入式的运行时收集（不需要下载到项目中）
- 将收集的数据分组可视化
- 用起来相对简单
- 方便进行结果数据分析报告的分享
- 最主要也是最大的优势就是它是免费的

## 缺点

大而全的工具并不能做到精细化深入的分析，在做 unity 性能优化时仍需要一些第三方工具或 unity 内置工具的辅助来进行更深入的分析。

但对于一款带优化项目来说，经常要做一些整体全面的分析迭代，并将这些分析结果作为历史保留，方便我们在后续迭代优化后做优化效果的对比。从这个方面来说，这个解决方案还是非常值得推荐的。

## 资源检查工具 asset checker

它可以对我们项目中使用到的资源做非侵入式检查，主要包括静态资源检查，包括检查资源的平台导入设置规格和合理性。

其次它可以对 asset bundle 进行分析，主要是分析包内有没有冗余资源。

此外它还可以对我们的静态代码进行分析检查

## 性能数据收集工具

分为桌面版与 app 版，桌面版要比 app 版功能更强大。

目前性能数据收集工具可以收集 unity profiler 数据，Unity Memory Profiler 数据，Render Dock 数据，特定 GPU 趋势数据等。

并在收集完成后生成可视化报告。


## UPR Tools

搭配 UPR 桌面版性能数据收集工具使用的软件包。

主要用来采集 load sense，load asset bundle，load asset，以及实例化数据。采集 load profiler 数据，收集 shader 变体数据，以及对我们项目的粒子资源进行性能分析和数据收集。

这里的很多数据是在编辑器下，或者打包过程中进行收集的，所以需要侵入式的集成此 package 到工程中。
